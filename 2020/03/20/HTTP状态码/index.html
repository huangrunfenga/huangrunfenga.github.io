HTTP状态码

发表于 2020-03-20 分类于 计网

一、一般情况

​ 1**：请求正被处理

​ 2**：请求成功处理

​ 3**：重定向啥的

​ 4**：客户端出错

​ 5**：服务端出错

二、常见状态码

  1. 200：请求已经被正常处理，常见于GET、POST

  2. 204：请求的 ’预检‘，在发起正式请求之前，先发一次预检，客户端询问服务端能不能接受接下来的正式请求，服务端回复204，表示可以而且不会返回响应实体，例子有cors复杂请求时候的option预检

  3. 206：partial Content，跟http请求头的‘Range’有关，该字段表示请求资源部分内容，有一个范围，

  4. 301：表示资源的永久重定向，服务端会返回新的资源地址，在响应头部的Loaction字段

  1. 302：表示资源是临时重定向，跟301差别就是一个临时一个永久
  2. 303：跟302很像，但是区别是在会提示客户在访问Location新的地址的时候要使用GET请求，实际的浏览器在处理301和302时，默认就会把原先的POST请求改为GET请求，所以实际上使用303的意义，单纯只是让语义化更清晰点。（303表示服务器明确告诉客户端，你要使用GET方法访问location;如果是302，就是仅仅告诉客户端要访问location,不限制方法，但是实际上客户端自己也会用GET方法访问。）
  3. 304：表示资源未改变，详见浏览器的协商缓存部分
  4. 307：跟重定向有关，前面的302默认会将访问Loaction的新地址的POST方法转为GET方法，307就是严格限制不允许浏览器将POST改为GET
  5. 400：对不起，你的请求存在语法错误，具体什么错误不知道，可能是请求方法错了，或者请求头多了少了什么东西
  6. 401：Unauthorized（未认证）用户登录之后会获得一个身份认证信息，然后生成mac之类的信息，放在请求头的Authorization字段里，发送给服务端，如果这个认证信息有问题或者根本没发送，就会出现这个状态码
  7. 403：禁止访问，服务器理解请求客户端的请求，但是拒绝执行此请求
  8. 404：服务器无法根据客户端的请求找到资源（网页），通常来说就是url错了
  9. 405：Method Not Allowed，客户端请求方法不被允许
  10. 407：跟401类似也要身份认证，但是请求资应该用代理授权
  11. 408：Request Time-out，服务端等待客户端请求时间过长，超时
  12. 409：发去PUT请求会返回的
  13. 410：也是客户端请求不到资源，但是和404不同，404是本来就没有，410是资源原来存在后来没用
  14. 416：对应206，表示请求资源的范围无效
  15. 414：Request-URI Too Large，请求的url过长
  16. 413：Request Entity Too Large，请求实体过大，服务器无法处理，因此拒绝请求。为防止客户端的连续请求，服务器可能会关闭连接。如果只是服务器暂时无法处理，则会包含一个Retry-After的响应信息
  17. 500：直接服务端出错，后台背锅
  18. 505：HTTP Version not supported，服务器不支持的HTTP版本
  19. 502：作为网关或者代理工作的服务器尝试执行请求时，从远程服务器接收到了一个无效的响应。这里说明下网关和代理服务器的概念：前面我们举例子都是直接客户端向服务器发请求，实际上客户端有时候不是直接向服务器请求，这中间可能存在网关和代理。
  20. 504：充当网关或代理的服务器，未及时从远端服务器获取请求，作为网关或者代理工作的服务器访问超时
